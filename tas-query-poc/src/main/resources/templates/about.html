About TAS Query Assistant

The TAS Query Assistant is an intelligent natural language interface for querying your Time and Attendance System database. It combines the power of AI-driven SQL generation with predictive analytics to provide insights into your workforce data.

Key Features

  • Natural language to SQL conversion
  • AI-powered query understanding with SQLCoder model
  • Predictive analytics and trend analysis
  • Interactive data visualizations
  • Conversation context awareness
  • Smart recommendations based on data patterns
  • Fallback pattern matching for reliability

Technology Stack

Spring Boot 3.x Spring AI 1.0.0 PostgreSQL Ollama SQLCoder 7B Thymeleaf Chart.js Gradle

How It Works

1. Natural Language Processing

When you type a question, the system uses the SQLCoder AI model to understand your intent and generate appropriate SQL queries. If AI is unavailable, it falls back to pattern matching.

2. Query Execution

The generated SQL is validated for security and executed against the PostgreSQL database containing your TAS data.

3. Insight Generation

Results are analyzed to extract key findings, trends, and anomalies. The system uses statistical analysis to identify patterns and generate recommendations.

4. Predictive Analytics

Advanced algorithms analyze historical data to predict future trends, identify risks, and suggest proactive measures.

5. Visualization

Data is automatically formatted for visual representation using Chart.js, with the appropriate chart type selected based on the data characteristics.

Database Schema

The system queries the tas_demo schema with the following main tables:

  • tenant - Organization/company information
  • location - Physical locations within tenants
  • colleague_details - Employee information
  • planned_shift - Scheduled work shifts
  • exception - Attendance exceptions (late, early, absence)
  • exception_detail - Detailed exception information
  • exception_audit - Exception resolution history

Privacy & Security

All queries are validated for SQL injection prevention. The system only allows SELECT queries and enforces schema restrictions. No data modifications are permitted through the chat interface.

AI processing is done locally using Ollama, ensuring your data never leaves your infrastructure.
